# dstack v0.3 — MCP Server + HTTP API

## Goal

Mirror all CLI commands as both HTTP API endpoints and MCP tools. Enables agent-to-agent integration, dashboard access, and hook automation.

## Architecture

New crate: `crates/dstack-server/` — Axum HTTP server + MCP stdio/SSE transport.

```
dstack-server/
├── src/
│   ├── main.rs         — Server entrypoint (axum + MCP)
│   ├── api.rs          — HTTP REST routes
│   ├── mcp.rs          — MCP tool definitions
│   └── lib.rs          — Shared server logic
└── Cargo.toml
```

Reuses `dstack-memory` and `dstack-cli` (as library) for all business logic. The server is a thin wrapper.

## HTTP API

```
GET  /health                          → { status: "ok" }
GET  /config                          → config summary
POST /memory/load    { project }      → [fields]
POST /memory/save    { key, value }   → { saved: true }
POST /memory/query   { pattern }      → [results]
GET  /memory/export                   → [all fields]
GET  /sync/status                     → [repo statuses]
POST /sync           { dry_run }      → [sync results]
POST /deploy         { service }      → { status }
POST /deploy/rollback { service }     → { status }
GET  /audit                           → summary
GET  /audit/stale                     → [stale docs]
GET  /skills                          → [available skills]
POST /skills/install { name }         → { installed }
POST /skills/sync                     → { installed, skipped }
```

## MCP Tools

Same operations as MCP tools (stdio or SSE transport):
- `dstack_memory_load`, `dstack_memory_save`, `dstack_memory_query`, `dstack_memory_export`
- `dstack_sync_status`, `dstack_sync`
- `dstack_deploy`, `dstack_deploy_rollback`
- `dstack_audit`, `dstack_audit_stale`
- `dstack_skills_list`, `dstack_skills_install`, `dstack_skills_sync`

## CLI Integration

```bash
dstack serve                          # start HTTP server (default :3500)
dstack serve --port 3500              # custom port
dstack serve --mcp                    # MCP stdio mode
dstack serve --mcp --transport sse    # MCP SSE mode
```

## Dependencies

- `axum` 0.8 — HTTP framework
- `tower-http` — CORS, tracing
- `tokio` — async runtime (already in workspace)
- Reuses `dstack-memory` and `dstack` (cli library) for all logic

## Security

- Reads config from same `~/.config/dstack/config.toml`
- `.env` auto-loading same as CLI
- Bind to localhost by default (127.0.0.1)
- No auth for v0.3 (local-only server)

## Out of Scope (v0.3)

- Authentication (future: API key or service key)
- Remote access (future: Caddy reverse proxy)
- WebSocket streaming (future: for live sync status)
